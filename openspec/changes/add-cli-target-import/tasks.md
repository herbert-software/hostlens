## 2. target-import — 提升 + 探测（TargetProbe）

- [x] 2.1 `build_one_target(entry, settings)`（**加在 `targets/registry.py`**，与 `build_registry_from_config` 同模块）:把单 entry 包成 1-entry `TargetsConfig` 复用 `build_registry_from_config(config, settings).get(name)`（construct+register **注入 `_entry`**，禁裸实例化）。+ `CandidateTarget → TargetEntry` 提升（`targets/probe.py`，import `build_one_target`）:映射 + `LocalEntry`/`SSHEntry` 校验,**提升后 SSHEntry.password 恒 None**（凭据仅经 env 引用透传,禁内联 `${VAR}`）;提升失败归 `invalid_candidate` 不抛。验收:合法 SSH 候选构造后 `exec` **不** raise `ssh_no_entry`；畸形候选 → `invalid_candidate` 整批不崩。
- [x] 2.2 `ProbeResult` 模型（脱敏标量 `reachable`/`capabilities: list[str]`(=`target.capabilities` 枚举投影,**非**探测命令输出)/`fingerprint: dict[str,str]`(**白名单键 `{os,kernel,arch,runtime}` 禁 hostname;值截断 ≤64 + 剥控制字符**,docker/podman/kubectl 探测进 `fingerprint.runtime` 非 capabilities)/`error_kind: str|None` **闭集 `{unreachable,auth_failed,timeout,exec_failed}`**，可 round-trip）+ `TargetProbe.probe`:先跑**固定字面量**只读命令（`cat /etc/os-release` 不 `source`）判 `reachable`(**`reachable := exec 返回 ExecResult 且 not timed_out`**;`timed_out=True`→reachable=False+`error_kind=timeout`→failed_probe;`exit_code≠0` 仍 reachable);**`TargetError.kind` 全映射表**（`ssh_connect_timeout→timeout`/`ssh_auth_failed→auth_failed`/`{ssh_connect_failed,ssh_connection_lost}→unreachable`/`{ssh_no_entry,target_disabled,未列}→exec_failed` fallback;`ExecResult.timed_out`→`timeout`）。验收:**非 root** 对 `local` 真探测 → **`reachable=True`(exec 返回且 not timed_out 判,不以 capabilities 非空判)**；**timed_out=True→reachable=False+failed_probe**；fingerprint 不含 hostname + 值截断;capabilities 来自 `target.capabilities`(podman 不进);error_kind 全映射闭集校验;JSON round-trip;命令固定字面量。
- [x] 2.3 `TargetProbe.probe_many` 并发 + 信号量限流（默认 ≤10，`concurrency` 可配）+ per-host timeout 失败隔离（`TargetError`/timeout → `error_kind` 枚举，**不**泄 host/user@host/traceback）。验收:连不上 target → `reachable=False`+脱敏枚举 error_kind，不抛不影响同批；**并发背压**:候选数 > max_concurrent 时在途受信号量约束、不崩；**脱敏测试**:error_kind 不含 host IP/凭据。
