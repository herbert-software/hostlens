## 2. systemd inspector 时间锚 + severity（病 1，`linux.systemd.failed_units`）

- [x] 2.1 `inspectors/builtin/linux/systemd_failed_units.yaml`：`version 1.0.0→1.1.0`；collect 一条 pipeline——`read -r uptime_seconds _ < /proc/uptime`（**shell 内建、不引入 `cut`**，`requires_binaries` 维持 `[systemctl, awk]`；**纯 POSIX，禁 `systime()`**，读不到 → exit 1）+ 列 failed 单元 + 每单元 `systemctl show -p Type -p InactiveEnterTimestampMonotonic`（per-unit 迭代用 **`while IFS= read -r unit`**、非 `for ... in $(...)`），emit JSON `{uptime_seconds, results:[{unit,type,inactive_monotonic_us}]}`（迭代键 `results`；`unit` 走引号+JSON-escape，**`uptime_seconds`/`inactive_monotonic_us` 必须 emit 为裸数字、不加引号**——否则 jsonschema int/number 校验失败→`status=exception`；**空集仍 emit** `{"uptime_seconds":N,"results":[]}`，`parse_json` 拒顶层数组）。
- [x] 2.2 output_schema：`required:[uptime_seconds, results]`，`uptime_seconds:number` + `results` 数组（`unit:str/type:str/inactive_monotonic_us:int`）；`parameters`：`boot_window_seconds`(default 180) / `min_uptime_seconds`(default 3600)。
- [x] 2.3 findings（`for_each: "results as u"`）：warning 当 `u.type=='oneshot' and u.inactive_monotonic_us>0 and u.inactive_monotonic_us<=boot_window_seconds*1000000 and uptime_seconds>=min_uptime_seconds`；critical 当上式取反（`when` 用属性式 `u.type`）。**message 用下标 `{u[unit]}` / `{u[type]}`**（`str.format` 不支持属性式 `{u.unit}`，会致格式化失败→finding 被丢；范式同 `containers_restart_loop.yaml`）；**warn+crit 两条 message 各保留 ≥1 CJK**（systemd 是 i18n `_MIGRATED_ALLOWLIST` 唯一成员，`test_message_contains_cjk` 逐 FindingRule 校验）；**只注入** `u[...]` 与 `output_schema.properties` 键（`uptime_seconds`），**禁止注入 parameter 名**（`boot_window_seconds`/`min_uptime_seconds` 不在 output_schema → 触发 `test_injected_fields_are_declared` 失败），不再引用已删的 `failed_names`。**接受 finding-id churn**。
- [x] 2.4 测试（D-7）：`_CaptureTarget` 编 `systemctl list-units`+per-unit `show`+`/proc/uptime` stdout、frozen monotonic、命令串级锁；断言 oneshot+开机窗口+`uptime>=min` → warning（病 1 锚：cloud-init 不拉整队 critical）、**fresh-reboot（低 uptime）oneshot → critical**、常驻/窗口外 → critical；**重写 `tests/inspectors/test_systemd_failed_collector.py`**（断言新 `results`/Type/monotonic JSON + **重建 unit 名 JSON-escape 回归**：含 `\`/`"` 的单元名仍合法 JSON）。
